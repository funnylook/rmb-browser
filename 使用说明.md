# RMB浏览器 — 使用说明

## 一、项目简介

基于 Android 原生 WebView 开发的轻量浏览器，支持：
- 📑 多标签浏览，无限标签页
- 🔒 独立会话，每个标签独立 Cookie 和登录态
- 🕶️ 无痕模式，单标签或全局无痕
- 🔧 开发者模式，支持 USB 远程调试
- 🔍 地址栏输入网址或搜索关键词
- 📱 适配澎湃3.0，Material3 动态取色

---

## 二、功能说明

### 1. 多标签浏览

- **新建标签**：点击底部工具栏 `+` 按钮
- **切换标签**：点击标签计数按钮，进入标签管理页
- **关闭标签**：在标签管理页点击 `×` 按钮
- **标签数量**：地址栏左侧显示当前标签数量角标
- **独立会话**：每个标签拥有独立的 WebView 实例，Cookie、Session、缓存完全隔离
- **不同账号**：不同标签可以同时登录同一网站的不同账号

### 2. 无痕模式

- **单标签无痕**：新建标签时选择"无痕标签"
- **全局无痕**：设置页开启"全局无痕模式"，所有新建标签默认无痕
- **无痕特性**：
  - 不保存 Cookie
  - 不保存表单数据
  - 不保存密码
  - 不使用缓存
  - 关闭标签时自动清除所有数据
- **清除无痕数据**：设置页可一键清除所有无痕标签及其数据

### 3. 开发者模式

- **远程调试已默认开启**
- 使用步骤：
  1. 手机连接电脑 USB
  2. 电脑 Chrome 浏览器打开 `chrome://inspect`
  3. 等待设备出现，选择要调试的页面
  4. 可使用 Elements、Console、Network、Sources 等所有 DevTools 功能
- **SSL 证书**：开发者模式下允许所有 SSL 证书错误，方便调试本地服务

### 4. 地址栏

- **输入网址**：直接输入 URL，自动补全 `https://`
- **搜索**：输入非网址内容，自动使用百度搜索
- **快捷键**：输入完成后按回车键（Go）直接跳转
- **加载状态**：地址栏下方显示加载进度条

### 5. 底部工具栏

| 按钮 | 功能 |
|------|------|
| ← | 后退（上一页） |
| → | 前进（下一页） |
| + | 新建普通标签 |
| 🕶️ | 新建无痕标签 |
| ⊞ | 打开标签管理 |

### 6. 标签管理页

- 显示所有已打开的标签
- 每个标签显示：图标（普通/无痕）、标题、URL
- 当前活跃标签高亮显示
- 点击标签切换到该标签
- 点击 `×` 关闭标签
- 底部快捷按钮：新建普通标签 / 新建无痕标签

### 7. 设置页

| 功能 | 说明 |
|------|------|
| 全局无痕模式 | 开关，新标签默认使用无痕模式 |
| 开发者模式 | 显示远程调试使用说明 |
| 清除无痕数据 | 关闭所有无痕标签并清除缓存 |
| 清除所有数据 | 关闭所有标签，清除所有 Cookie、缓存和历史 |
| 关于 | 版本信息和功能介绍 |

## 三、技术特性

### WebView 配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| JavaScript | 启用 | 支持所有网页交互 |
| DOM Storage | 启用 | 支持 localStorage/sessionStorage |
| Database | 启用 | 支持 Web SQL |
| 缓存模式 | 普通/无缓存 | 无痕模式不使用缓存 |
| 缩放 | 启用 | 支持手势缩放，隐藏缩放控件 |
| 混合内容 | 兼容模式 | HTTPS 页面可加载 HTTP 资源 |
| 文件访问 | 禁用 | 安全考虑，禁止访问本地文件 |
| 远程调试 | 启用 | 支持 Chrome DevTools |

### 会话隔离机制

每个标签页拥有独立的：
- WebView 实例
- Cookie 存储
- Session 状态
- 缓存数据
- 表单历史

这意味着：
- 同一网站不同标签可以登录不同账号
- 无痕标签关闭后数据自动清除
- 普通标签的数据在应用关闭后保留

### SSL 处理

- 开发者模式下允许所有 SSL 证书错误
- 方便调试本地 HTTPS 服务或自签名证书
- 生产环境建议根据需要调整

## 四、数据存储

| 数据 | 存储位置 | 说明 |
|------|----------|------|
| Cookie | WebView 内部存储 | 普通标签持久化，无痕标签不保存 |
| 缓存 | WebView 内部存储 | 普通标签使用缓存，无痕标签不使用 |
| 历史记录 | WebView 内部存储 | 浏览器前进/后退历史 |
| 标签状态 | 内存 | 应用关闭后标签状态丢失 |

- 所有数据仅存储在本地
- 无痕模式不保存任何数据
- 可通过设置页一键清除

## 五、权限说明

| 权限 | 用途 |
|------|------|
| `INTERNET` | 访问网络，加载网页 |
| `ACCESS_NETWORK_STATE` | 检测网络状态 |
| `WRITE_EXTERNAL_STORAGE` | 下载文件（Android 9 以下） |
| `READ_EXTERNAL_STORAGE` | 读取下载文件（Android 12 以下） |
| `CAMERA` | 网页摄像头功能（如视频通话） |

## 六、编译与安装

### 使用 Android Studio

1. 下载并安装 [Android Studio](https://developer.android.com/studio)
2. 打开项目文件夹 `rmb-browser`
3. 等待 Gradle Sync 完成
4. 点击 Build → Build APK
5. 输出路径：`app/build/outputs/apk/debug/app-debug.apk`

### 使用命令行

```bash
cd rmb-browser
chmod +x gradlew
./gradlew assembleDebug
```

### GitHub Actions 自动编译

1. 推送代码到 GitHub `main` 分支
2. GitHub Actions 自动开始编译
3. 编译完成后在 Actions → Artifacts 下载 APK

## 八、常见问题

**Q: 为什么需要这么多权限？**
A: `INTERNET` 是必须的。其他权限是可选的，用于下载文件、摄像头等网页功能。不授权不影响基本浏览。

**Q: 无痕模式和普通模式有什么区别？**
A: 无痕模式不保存 Cookie、缓存、表单数据、密码。关闭标签后所有数据自动清除。普通模式数据会保留。

**Q: 如何同时登录同一网站的多个账号？**
A: 新建多个标签页，每个标签页独立会话，可以在同一网站登录不同账号。

**Q: 远程调试怎么用？**
A: 手机 USB 连接电脑，电脑 Chrome 打开 `chrome://inspect`，等设备出现后选择页面即可调试。

**Q: 为什么有些网页显示不正常？**
A: WebView 兼容性不如完整浏览器。部分依赖特定浏览器 API 的功能可能不支持。

**Q: 如何清除所有浏览数据？**
A: 进入设置页 → 清除所有数据，会关闭所有标签并清除所有 Cookie、缓存和历史。

**Q: 支持哪些 Android 版本？**
A: 最低支持 Android 8.0（API 26），推荐 Android 12+ 以获得最佳动态取色效果。

**Q: 应用体积多大？**
A: 约 2-3MB，非常轻量。使用系统 WebView 渲染，不内置浏览器内核。
